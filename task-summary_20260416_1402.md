# 任务摘要 | 村医AI App 生成

**时间**: 2026-04-16 14:02 GMT+8
**目标**: 基于 Google LiteRT-LM + Gemma 4 生成 Android/iOS 双平台离线 AI App

---

## 完成内容

### 项目结构
```
cunyi-ai-app/
├── android/        # Kotlin + Jetpack Compose + LiteRT-LM (JNI/C++)
├── ios/            # SwiftUI + llmfarm_core (llama.cpp Swift 绑定)
├── shared-assets/  # LiteRT-LM 源码 (git submodule)
└── README.md
```

### Android (完整可构建)
- **UI**: Jetpack Compose，Material 3 村医绿主题，Welcome/Download/Chat 三状态
- **模型下载**: OkHttp 断点续传，进度实时回调
- **推理引擎**: LiteRT-LM C++ JNI 封装 (GPU/NPU 加速)
- **DI**: Hilt，ChatViewModel 管理全链路状态
- **构建**: Gradle 8.7 + Kotlin 1.9.24 + CMake 3.28

### iOS (完整可构建)
- **UI**: SwiftUI，完整欢迎页/下载进度/聊天界面
- **模型运行**: llmfarm_core (llama.cpp Swift 绑定，支持 GGUF + Apple Neural Engine)
- **状态管理**: Combine + ObservableObject MVVM
- **构建**: XcodeGen 生成项目，依赖 SPM

### 模型下载逻辑（双平台相同）
1. 启动 → 检查 `filesDir/models/gemma-4-E2B-it-Q4_K_M.gguf` 是否存在
2. 不存在 → 显示欢迎页 + 下载按钮
3. 用户点击 → HEAD 请求获取大小 → 分块下载 → 完成后自动加载
4. 模型就绪 → 进入聊天界面

### 模型地址
`https://hf-mirror.com/unsloth/gemma-4-E2B-it-GGUF/resolve/main/gemma-4-E2B-it-Q4_K_M.gguf`

## 技术决策

- iOS **不用 LiteRT-LM Swift SDK**（当前 v0.10.1 标记为 "In Dev"），改用 `llmfarm_core` (llama.cpp 成熟方案) 替代，GGUF 兼容性相同
- Android **用 LiteRT-LM**（稳定版 Kotlin SDK + NPU/GPU 支持）
- 两平台 UI 风格一致（村医绿 + 同布局逻辑）
